# Codebase Architecture Guide

## Executive Summary

The Layer7 codebase implements a sophisticated layered architecture for parsing, structuring, and executing code-based workflows. Data flows through a well-defined pipeline where `Layer7Parser` in `core_structure.py` processes text input into structured `HeaderNode` objects, which are then organized into a hierarchical tree structure. The `AddressResolver` in `addressing.py` manages identifier resolution and path mapping, while `CompositionEngine` in `flow_control.py` orchestrates execution flows based on routing rules. The system maintains state through various internal variables and class attributes, with `MCPDispatcher` in `language_integration.py` handling cross-language execution and variable management.

The core state management relies on a combination of class-based state tracking and internal variable management. The `Layer7Parser` maintains mutable state through variables like `current_node`, `in_code_block`, and `code_lines` during parsing operations. The `CompositionEngine` manages execution state through `state`, `stdin_data`, and `result` variables, while `MCPDispatcher` maintains temporary file handles and execution contexts. All state transitions are managed through method calls that modify these internal variables, creating a tightly-coupled but predictable state machine.

Hidden architectural coupling exists primarily through the tight integration between `Layer7Parser` and `AddressResolver` dependencies, where `AddressResolver`'s `resolve` method directly manipulates `HeaderNode` objects created by `Layer7Parser`. Additionally, `MCPDispatcher` has implicit dependencies on both `Layer7Parser`'s output structure and `AddressResolver`'s identifier resolution, creating a complex web of interdependencies that makes the system difficult to refactor or test in isolation. The `CompositionEngine` also exhibits coupling through its reliance on specific data structures and state variables from both `core_structure.py` and `language_integration.py`.
