# Codebase Architecture Guide

The system operates as a pipeline-driven document-to-execution engine, beginning with `Layer7Parser` ingesting raw text and structuring it into a hierarchical `HeaderNode` tree. This parsed topology is subsequently processed by `AddressResolver`, which normalizes identifiers and resolves cross-references to establish a coherent dependency graph. Control flow is then extracted and orchestrated by `CompositionEngine`, which interprets routing rules, step sequences, and conditional logic to direct execution. Finally, `MCPDispatcher` consumes these resolved execution paths, dynamically generating language-specific wrappers and managing temporary artifacts to invoke external interpreters, ultimately returning structured `ExecutionResult` objects back through the pipeline.

State management is strictly instance-scoped and phase-specific, avoiding global mutable stores in favor of explicit context tracking. During parsing, `Layer7Parser` maintains transient state variables (e.g., `in_code_block`, `current_node`, `code_lines`) to track structural boundaries and metadata extraction. At runtime, `CompositionEngine` manages execution state through a cursor-driven model, tracking `current_val`, routing rule matches, and step progression to handle loops and conditionals deterministically. `MCPDispatcher` complements this by managing ephemeral execution state, including temporary file paths, interpreter contexts, and cleanup routines, ensuring that each composition run remains isolated and garbage-collected upon completion.

Despite a modular file structure, several implicit couplings threaten long-term maintainability. `CompositionEngine` tightly couples control-flow parsing with execution logic, embedding routing validation and step traversal directly within the execution lifecycle rather than delegating to a dedicated interpreter or AST evaluator. Furthermore, `MCPDispatcher` exhibits significant language-specific coupling through hardcoded preamble generation and temp-file orchestration, creating a maintenance bottleneck as new languages are added. The addressing and parsing layers also share implicit contracts regarding node path normalization and identifier resolution, which are not enforced via interfaces, risking brittle failures if the document schema evolves. Decoupling the routing engine from execution and abstracting the language dispatch layer behind a strategy pattern would significantly improve testability and extensibility.
