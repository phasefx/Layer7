# Codebase Architecture Guide

### Executive Summary

The system operates as a multi-stage pipeline that transforms unstructured text into executable logic. The process begins with the `Layer7Parser` in `core_structure.py`, which decomposes input into a hierarchical tree of `HeaderNode` objects. Once structured, the `AddressResolver` in `addressing.py` provides a mechanism for semantic referencing and navigation of these nodes via normalized identifiers. This structured data is then ingested by the `CompositionEngine` in `flow_control.py`, which applies routing rules and control flow logic—such as loops, conditionals, and jumps—to determine the execution path. Finally, the `MCPDispatcher` in `language_integration.py` translates these logical steps into concrete execution commands, wrapping them in language-specific preambles to interface with external interpreters.

State management is primarily driven by a hierarchical, tree-based model where the `HeaderNode` serves as the fundamental unit of state. This tree structure maintains parent-child relationships and metadata (such as code content, language types, and data values) that define the system's structural context. During execution, the `CompositionEngine` maintains a transient operational state, tracking the current value, cursor position, and active routing rules. This creates a dual-layer state model: a persistent structural state represented by the node tree and a volatile execution state managed during the traversal of composition chains.

Architecturally, there is significant implicit coupling between the `Layer7Parser` and the `CompositionEngine`. The engine's ability to route and execute logic is strictly dependent on the specific schema and identifier normalization logic implemented in `addressing.py` and `core_structure.py`. Any modification to the `HeaderNode` topology or the `normalize_identifier` logic would necessitate a simultaneous update to the `RoutingRule` logic to prevent execution failure. Furthermore, the `MCPDispatcher` introduces a heavy dependency on the semantic integrity of the parsed nodes, as it relies on the `code_lang` and `data_value` attributes to construct valid, executable environments.
