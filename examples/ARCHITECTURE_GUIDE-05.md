# Codebase Architecture Guide

The data flow in this codebase centers around parsing structured documentation or configuration files into an in-memory hierarchical representation, then executing or composing logic based on that structure. Input text is first processed by `Layer7Parser` in `core_structure.py`, which tokenizes headers, code blocks, and metadata (e.g., arrows, inline variables) into `HeaderNode` instances. These nodes are then registered with an `AddressResolver` in `addressing.py`, which builds a navigable tree with normalized identifiers and path resolution logic. Flow control directives—such as `RoutingRule`s in `flow_control.py`—are parsed from the same source and used by `CompositionEngine` to orchestrate execution chains, potentially routing between steps conditionally or repeatedly. Execution of code blocks is delegated to `MCPDispatcher` in `language_integration.py`, which dynamically wraps and executes code in subprocessed interpreters, returning structured results.

State management is handled through a combination of mutable instance variables and explicit context propagation. `AddressResolver` maintains a registry of nodes and supports dynamic registration and resolution via `register_node()` and `resolve()`, using normalized naming conventions (e.g., `normalized_leaf`, `full_normalized_name`) to avoid ambiguity. `CompositionEngine` holds runtime state like `cursor`, `steps`, and `routing_rules`, and uses `execute_chain()` to mutate internal state (`current_val`, `state`, `stdin_data`) as it processes steps. Crucially, `MCPDispatcher` encapsulates execution state via temporary file generation (`_write_temp`) and variable preamble injection (`_generate_var_preamble`), ensuring isolated and reproducible runs. The `ExecutionResult` class standardizes output, while `Layer7Error` provides a unified exception model.

A hidden architectural coupling exists between `flow_control.py` and `language_integration.py`: `CompositionEngine.execute_chain()` directly invokes `MCPDispatcher.execute()` (via `execute()` method), passing raw code strings and state, but without a formal interface or abstraction layer. This tight coupling implies that changes to `MCPDispatcher`’s signature or behavior (e.g., async execution, new result formats) will ripple into `flow_control.py`. Additionally, `AddressResolver` and `Layer7Parser` share implicit assumptions about node identity and path semantics—e.g., `HeaderNode.path` is used by `AddressResolver.register_node()`—but no explicit contract is enforced, risking inconsistencies if node metadata is malformed or incomplete. This coupling is further exacerbated by the use of global-like variables (e.g., `path_titles`, `leaf`, `parents`) in `addressing.py`, which suggest a shared conceptual model but lack encapsulation or validation.
