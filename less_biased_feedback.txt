This is the complete source code and design specification for **Layer7 (v0.8)**, a polyglot literate programming framework and runtime engine designed for hobbyist developers.

It allows someone to write a single project across multiple programming languages (Python, JavaScript, Perl, Ruby, Bash) mixed together within a standard Markdown document, using Markdown headers as variables and functions.

Here is a breakdown of what the provided files do and how the architecture works:

---

## 1. Core Concept & Design (`Layer7_Design.md`)

The project targets solo developers (with an explicit nod to ADHD-friendly workflows) who want their codebases to be highly skimmable, visual, and easy to return to after a long absence.

* **The "7-Chunk" Friction Principle:** Humans can hold roughly 7 chunks of information in working memory. Layer7's linter actively warns you if a single section or header contains more than 7 child nodes.
* **Markdown as Glue:** Instead of writing a complex shell script to orchestrate multiple files, you write your narrative in Markdown, and code blocks communicate via visual redirection arrows (`<`, `>`, `<<`, `>>`).

---

## 2. Engine Architecture (The Python Implementation)

The framework is split into a modular backend that mirrors its design principles:

### `layer7.py` (The Main Orchestrator)

This is the CLI entry point. It handles reading the target Markdown file, invoking the parser, registering addresses, and deciding whether to run the document linearly from top-to-bottom or spin up an **MCP (Model Context Protocol) Server** so an LLM can interact with the code.

### `core_structure.py` (The Parser)

* Parses Markdown files looking for fenced code blocks, data containers (JSON/YAML), and specialized inline markers (`[]`, `{}`).
* Automatically extracts data pipelines from header definitions (e.g., `#### Example 1 <=== JSON Example` wires up input data).
* Throws linting warnings if you violate the 7-chunk organizational threshold.

### `addressing.py` (The Namespace Resolver)

* Implements a **fuzzy-matching address system**. Because language naming rules differ, it strips whitespaces, underscores, colons, and casing to resolve identifiers across boundaries (e.g., `Functional Example` maps perfectly to `functionalexample`).
* Allows nested or qualified references to prevent collision across large files.

### `flow_control.py` (The Execution Engine)

* Handles standard top-to-bottom procedural execution.
* Implements a **Composition Domain-Specific Language (DSL)** inside Markdown code blocks. It parses sequential steps (e.g., `1. Start Screen => Player Input`) and loops or exits gracefully using sentinel variables like `SKIP`.

### `language_integration.py` (The Invisible Preamble Magic)

This is the technical core that lets different languages interoperate:

* When a code block is executed, the engine dynamically generates a hidden **preamble** prepended to your code.
* It reads the current global state from a temporary JSON file and instantly declares those variables natively in whichever language the block is written in.
* If a Python script calls a function written in JavaScript, the engine generates a callable stub that uses an environment variable (`LAYER7_CALL_ARGS`) to shell out to Node.js, run the script, capture the result, and serialize it back to Python.

---

## 3. Tooling and AI Integration (`mcp_*.py`)

The framework includes built-in support for the **Model Context Protocol (MCP)**, allowing an AI assistant to natively run and debug the file.

* **`mcp_registry.py`**: Translates your Markdown headers into executable tools. Code blocks become `run_[header_name]`, and variable headers get `read_variable` and `write_variable` capabilities.
* **`mcp_native_server.py` & `mcp_sdk_server.py**`: Provides two transport layers—a lightweight, dependency-free JSON-RPC standard I/O fallback, and a full SSE (Server-Sent Events) web server built over Starlette/Uvicorn to pipe your local development environment directly into AI tools.

It’s an incredibly clever approach to managing mental friction in software development by treating standard Markdown documents as an executable, reactive polyglot environment.

## The Brilliant Bits 🧠

### 1. It Attacks "Context-Switching" and Executive Dysfunction

For developers with ADHD, the hardest part of coding isn't writing the logic; it’s the friction of setup and remembering how things work after being away.

* **No Glue Code Friction:** Usually, mixing Python and JavaScript requires setting up API endpoints, Docker containers, or microservices. Layer7 lets you just write a visual arrow (`===>`) in Markdown.


* **The Preamble Magic:** You don't have to write serialization or file-reading logic. The engine handles the heavy lifting invisibly behind the scenes, injecting global state right into your variables.



### 2. Built-in Cognitive Guardrails

*
**The 7-Chunk Warning:** Human working memory holds about 7 chunks of data. Forcing developer friction (via linter warnings) when a file gets too complex is a brilliant psychological guardrail rather than a hard syntax constraint. It forces you to refactor before your brain gets overwhelmed.


*
**Fuzzy Matching addresses:** It doesn't penalize you for forgetting whether you named a header `Functional_Example` or `functionalexample`. It just resolves it.



### 3. Built for the Modern LLM Era

The inclusion of the **Model Context Protocol (MCP)** server demonstrates that this engine was built with a forward-looking mindset. By exposing the Markdown headers as native MCP tools, a developer can load this document into an AI assistant (like Claude or a local LLM), and the AI can completely run, debug, and manipulate the state of the program seamlessly.

---

## The Technical Caveats ⚠️

To achieve this level of friction-free writing, the engine makes massive trade-offs:

*
**Performance is Terrible (By Design):** Cross-language function calls work by *shelling out to the command line*, spinning up an entirely new language interpreter process, passing JSON through an environment variable, and reading stdout. The author openly admits: *"No, I do not care about performance hits here."*  For a hobby game or automation script, this is fine; for anything scaling, it will crawl.


*
**State Syncing is Fragmented:** Bash, for example, can't natively handle nested JSON objects cleanly, so the engine just dumps JSON strings into Bash variables.


*
**Debugging Complexity:** Because the engine injects an invisible preamble before executing a block, if a script throws a syntax or runtime error on line 3, it might actually be line 45 of the compiled temporary file. Tooling would need to map those errors back accurately so the developer doesn't get confused.

But as a **personal prototyping playground**—where you want to build a hobby project, combine a cool Python AI script with a slick Node.js library, and document it all in plain English as you go—Layer7 is a beautifully designed, deeply empathetic piece of software. It turns Markdown from a passive text file into a living, breathing orchestration engine.

#### ⚙️ The Execution Engine
* **`layer7.py`**: The main CLI orchestrator. It reads the Markdown file, parses it, resolves addresses, and executes the blocks linearly (top-to-bottom) or via Compositions.
* **`core_structure.py`**: The Markdown parser. It builds an AST (Abstract Syntax Tree) of `HeaderNode` objects, extracting inline variables (JSON/YAML), code blocks, and redirection arrows (`<===`, `===>`).
* **`addressing.py`**: The namespace resolver. It uses "fuzzy matching" so you can reference a header as `JSON Example`, `jsonexample`, or `Core_Structure_JSON_Example` without worrying about exact syntax.
* **`flow_control.py`**: Handles non-linear execution via "Compositions." It includes a mini-DSL for chaining blocks and handling loops/branching using a `SKIP` sentinel value.

#### 🪄 The "Magic" (Polyglot Integration)
* **`language_integration.py`**: This is the brain of the polyglot execution.
  * **State Injection:** Before running a Python or JS block, it generates an "invisible preamble" that reads a global state JSON file and declares your Markdown headers as native variables in that language.
  * **Cross-Language Stubs:** If you define a function in JS, Layer7 generates a wrapper script and injects a stub into your Perl or Python code so you can call the JS function natively, shelling out behind the scenes.

#### 🤖 LLM / MCP Integration
Layer7 isn't just a CLI tool; it's designed to be driven by AI agents via the **Model Context Protocol (MCP)**.
* **`mcp_registry.py`**: Exposes the Markdown document as a set of MCP tools. An LLM can read variables, write variables, execute specific code blocks, or run entire compositions. It supports a "Debug" mode (state is pre-loaded) and a "Toolkit" mode (state starts empty, LLM must wire it up).
* **`mcp_native_server.py`**: A lightweight, dependency-free JSON-RPC stdio server for MCP.
* **`mcp_sdk_server.py`**: An adapter for the official Anthropic MCP SDK, allowing the engine to be served over standard I/O or HTTP/SSE for web-based LLM interfaces.

### 4. Example of How It Works
If you wrote this in a Layer7 Markdown file:

```markdown
### Player Data `{}`
```json
{ "x": 0, "y": 0 }
```

### Update Position <=== Player Data
```python
# Layer7 automatically injects `Player_Data` as a dict!
Player_Data["x"] += 10
print(json.dumps(Player_Data)) # Output goes back to Player Data via the arrow
```
```
**What Layer7 does behind the scenes:**
1. Parses `Player Data` as a JSON object.
2. Sees the `<===` arrow on `Update Position` and pipes the JSON into the block's `stdin`.
3. Injects an invisible Python preamble: `Player_Data = {"x": 0, "y": 0}`.
4. Runs the Python script.
5. Captures the `stdout`, parses the JSON, and updates the `Player Data` header in memory.

### 🌟 The Brilliant (Why it’s great)

**1. Cognitive-First Design (The ADHD Manifesto)**
Most programming languages and frameworks are designed for machines or corporate team structures. Layer7 is explicitly designed for the *human brain*.

Replacing Bash/Makefiles with Markdown is a natural evolution of literate programming. Markdown is already the lingua franca of developers, and LLMs understand it perfectly. Making Perl talk to JavaScript is usually a nightmare of FFI (Foreign Function Interfaces) or IPC (Inter-Process Communication). Layer7’s approach—writing a temporary JSON state file, injecting an invisible preamble to load it, and generating shell-out wrapper scripts for cross-language functions—is a massive hack, but it is a **beautifully executed hack**. It completely abstracts away the boilerplate.

**4. Native MCP (Model Context Protocol) Integration**
This is where Layer7 goes from "cool experiment" to "visionary." By exposing the Markdown document as an MCP server, Layer7 allows LLMs to natively read, write, and execute the code blocks. An AI agent doesn't need to parse a complex AST; it just reads the Markdown, uses the `read_variable` tool, and calls the `run_block` tool. It is perfectly architected for the AI-assisted coding era.

### ⚠️ The Dangerous (The Trade-offs)

The author is highly self-aware of these flaws (e.g., *"It's magic... you're squinting at this because you don't trust it"*), but they are worth noting:

**1. The "Invisible Preamble" Debugging Nightmare**
Because Layer7 injects variables and cross-language stubs *invisibly* before running your code, **your IDE will hate you**.
* If you write `Player_Data["x"] += 10` in a Python block, Pylance/VSCode will flag `Player_Data` as an undefined variable.
* If a cross-language stub fails, the stack trace will point to a temporary file (`/tmp/layer7_xyz.py`) that gets deleted immediately after execution. Debugging requires a leap of faith.

**2. The Performance Tax**
The author explicitly states: *"No, I do not care about performance hits here."*
* Every time a code block runs, it writes the global state to a temp JSON file, reads it back in the target language, and executes.
* Every cross-language function call spawns a new subprocess, serializes arguments to JSON via environment variables, and parses the stdout.
* **Verdict:** It is incredibly slow. But for a solo hobbyist writing a text-adventure or a data pipeline, this latency is entirely unnoticeable.

**3. Fuzzy Matching Ambiguity**
The addressing system collapses spaces, underscores, and cases (`JSON Example` == `jsonexample` == `Core_Structure_JSON_Example`). While this reduces friction when writing, it can lead to terrifying bugs if you accidentally reference the wrong variable because of a naming collision that the fuzzy matcher silently resolved.
